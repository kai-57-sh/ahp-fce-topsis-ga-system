# 作战场景目录
# Operational Scenarios Directory

## 概述

本目录包含AHP-FCE-TOPSIS-GA评估系统的四类典型海上作战场景，涵盖了现代海上作战的主要类型。每个场景都经过精心设计，反映了真实的作战环境和威胁条件。

## 场景分类

### 1. 近岸水下侦察监视 (`nearshore_underwater_recon.yaml`)

**场景描述**: 在近岸复杂水下环境中执行持续水下侦察监视任务

**主要特点**:
- 地理环境: 复杂近岸海域，水深20-200米
- 威胁类型: 敌方潜艇、水下设施、小型UUV
- 任务重点: 水下监视、目标识别、威胁预警
- 典型平台配置: 12-25个无人平台，强调隐身和被动侦察能力

**适用评估目标**:
- 侦察探测体系效能
- 水下监视覆盖能力
- 威胁预警响应速度

### 2. 海峡要道控守 (`strait_control_defense.yaml`)

**场景描述**: 在关键海峡要道实施全方位控制，阻止敌方通过

**主要特点**:
- 地理环境: 海峡要道，宽度15-50公里
- 威胁类型: 水面战斗群、潜艇、导弹攻击艇、航空威胁
- 任务重点: 通道控制、多层防御、火力打击
- 典型平台配置: 20-40个无人平台，强调打击和防御能力

**适用评估目标**:
- 区域控制防御能力
- 多层防御体系效能
- 火力打击精确性

### 3. 登陆场通道清扫 (`amphibious_landing_clearance.yaml`)

**场景描述**: 为两栖登陆作战清扫安全通道，清除水雷和障碍物

**主要特点**:
- 地理环境: 登陆海滩及周边海域，水深梯度变化
- 威胁类型: 水雷障碍、水下障碍物、岸防火力
- 任务重点: 通道开辟、水雷清除、障碍破除
- 典型平台配置: 25-45个无人平台，强调清障和作业能力

**适用评估目标**:
- 水雷障碍清除效率
- 登陆通道保障能力
- 工程作业协调能力

### 4. 要害目标封锁 (`high_value_target_blockade.yaml`)

**场景描述**: 对敌方重要港口、基地实施海上封锁

**主要特点**:
- 地理环境: 港口基地周边，半径50-100公里封锁区
- 威胁类型: 护航编队、潜艇、商船威胁、空中威胁
- 任务重点: 海上封锁、目标拦截、持续维持
- 典型平台配置: 28-50个无人平台，强调监视和拦截能力

**适用评估目标**:
- 封锁行动有效性
- 目标识别拦截能力
- 长期持续作战能力

## 场景使用指南

### 选择合适的场景

根据评估目标和作战需求选择场景：

1. **侦察监视系统评估** → 近岸水下侦察监视
2. **防御体系效能评估** → 海峡要道控守
3. **工程保障能力评估** → 登陆场通道清扫
4. **封锁行动效果评估** → 要害目标封锁

### 使用场景文件

```bash
# 使用特定场景进行方案评估
python main.py evaluate --schemes data/schemes/your_scheme.yaml --scenario data/scenarios/operational/nearshore_underwater_recon.yaml

# 使用特定场景进行GA优化
python main.py optimize --scenario data/scenarios/operational/strait_control_defense.yaml --population 50 --generations 100
```

### 场景参数调整

每个场景文件都可以根据具体需求调整：

1. **威胁环境**: 修改威胁类型、数量、能力
2. **任务目标**: 调整目标权重和成功标准
3. **平台配置**: 修改基因编码范围和约束
4. **评估指标**: 调整评估指标侧重和权重

## 场景定制开发

如需开发新的作战场景，请参考现有场景的结构：

1. **基本信息**: 场景ID、名称、类型、描述
2. **环境设定**: 地理、威胁、任务持续时间
3. **任务目标**: 主要目标和权重分配
4. **基因编码**: 平台类型和数量范围
5. **约束条件**: 预算、能力、部署限制
6. **评估重点**: 指标侧重和权重配置
